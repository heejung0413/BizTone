## [기획서] AI 비즈니스 커뮤니케이션 에이전트

### 1. 개요

- **목적:** 실무자가 작성한 핵심 내용(Raw Data)을 바탕으로, 상황(채팅, 메일, 보고 등)에 최적화된 비즈니스 텍스트로 변환.
- **핵심 가치:** 커뮤니케이션 비용 감소, 프로페셔널한 이미지 구축, 상황별 에티켓 준수.

---

### 2. 주요 기능 (Core Features)

1. **초안 입력 (Raw Input):** 개조식, 불완전한 문장, 단순 사실 나열 등 형식에 구애받지 않는 입력.
2. **포맷 선택 (Format Selection):** \* **Chat:** 협업 툴(Slack, Teams 등)용. 간결, 두괄식, 위트 섞인 어조.
   - **Email:** 비즈니스 메일용. 격식, 정중함, 제목 생성 기능 포함.
   - **Casual:** 사내 메신저, 사적 대화용. 부드럽고 친절한 어조.
   - **Report:** 상사 보고용. 요약 중심, 개조식, 성과 중심 단어 선택.
3. **톤 조절 (Tone Adjustment):** (친절하게 / 단호하게 / 정중하게) 등 상세 옵션 추가.
4. **역할 설정 (Persona):** (팀원으로서 / 팀장으로서 / 외부 업체 담당자로서) 역할 기반 작성.

---

### 3. 개발 아키텍처 및 기술 스택

- **LLM 엔진:** GPT-4o 또는 Claude 3.5 Sonnet (문맥 파악 및 한국어 자연어 처리에 최적화).
- **프롬프트 기법 (Prompt Engineering):** \* **Few-Shot Learning:** 각 포맷별 '좋은 예시' 데이터를 미리 주입하여 결과물 품질 향상.
  - **Role Prompting:** 시스템 프롬프트에 "너는 10년 차 IT 기업 홍보 전문가야" 등 페르소나 부여.
- **Frontend:** React + Tailwind CSS (빠른 UI 구현 및 모바일 대응).
- **Backend:** Node.js (Express/FastAPI) + OpenAI API / LangChain 활용.

---

### 4. 핵심 데이터 흐름 (Data Flow)

1. **사용자 데이터:** "회의 시간 2시에서 3시로 변경됨, 장소는 회의실A 그대로" 입력.
2. **에이전트 처리:** 선택한 포맷(Email)과 페르소나(팀장)에 맞는 프롬프트 조합.
3. **LLM 생성:** > **[Email 예시]** "안녕하세요. 오늘 예정된 회의 시간이 불가피하게 변경되었습니다. (기존 2시 -> 변경 3시) 장소는 동일하게 회의실A입니다. 일정에 참고 부탁드립니다."
4. **출력 및 수정:** 사용자가 확인 후 '복사' 또는 '다시 생성'.

---

### 5. 차별화 포인트 (Unique Selling Point)

- **히스토리 관리:** 자주 쓰는 문구 저장 및 이전 변환 기록 확인.
- **익스텐션 연동:** **크롬 익스텐션(MV3)**으로 개발하여 Gmail, Slack 웹 버전에서 바로 팝업을 띄워 작성 지원. (질문하신 MV3 지식을 여기서 활용!)
- **데이터 보안:** 입력한 실무 내용이 외부 학습 데이터로 활용되지 않도록 하는 프라이버시 설정.

---

### 6. 향후 확장성

- **다국어 대응:** 한글로 쓴 초안을 즉시 정중한 영어/일어 메일로 번역 및 작성.
- **AI 자동 제안:** 입력된 내용이 '보고' 성격인지 '공지' 성격인지 AI가 판단하여 포맷 자동 추천.
